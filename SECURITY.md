# Security Policy

SectorForge is a local-first telemetry app. Please help keep reports private until maintainers have had a chance to investigate and publish a fix.

## Reporting A Vulnerability

Do not publish exploit details, proof-of-concept code, private telemetry captures, secrets, or local game paths in a public issue.

Use GitHub private vulnerability reporting for this repository when it is available. If private reporting is not available, open a public issue that asks for a private maintainer contact and includes only a high-level description, such as the affected component and impact category.

Helpful private reports include:

- The affected version, branch, or commit.
- The impacted component, such as API, collector, storage, web dashboard, or adapter code.
- A concise description of the risk and expected impact.
- Safe reproduction steps that avoid sharing secrets, captures, or exploit-ready details.
- Any mitigations or workarounds you have already tried.

Maintainers will acknowledge valid reports when possible, investigate the issue, and coordinate a fix or disclosure plan before public details are shared.

## Supported Versions

SectorForge is early-stage software. Security fixes are expected on the default branch until formal releases are published.
